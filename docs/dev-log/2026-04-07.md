## 의사결정

### draft 생성 위치

- 고민:
  - 노트 선택 시 draftNote를 생성할 위치(컴포넌트)

- 선택지:
  1. NoteEditor
  2. NotePage

- 판단 기준:
  - 원본 상태와 편집 상태의 분리
  - 컴포넌트 책임 범위

- 결정:
  - NotePage에서 원본을 복사해 draft 생성 후 전달

- 이유:
  - Editor는 원본 상태를 직접 다루지 않고 draft만 수정하도록 제한
  - 이를 통해 원본 데이터의 불필요한 변경을 방지하고
  - 저장 시점에만 상태 변경이 일어나도록 제어하기 위함

- 고려했지만 선택하지 않은 이유:
  - Editor에서 draft를 생성할 경우,
    원본 데이터 접근 책임이 Editor로 내려가
    컴포넌트 역할이 불명확해질 수 있음
