## 의사결정

### draft 완성 위치

- 고민:
  - 노트 수정 시 draftNote를 완성할 위치(컴포넌트)

- 선택지:
  1. NoteEditor
  2. NotePage

- 판단 기준:
  - 컴포넌트 책임 범위
  - 재사용성

- 결정:
  - patch를 전달받아 Page에서 draft 조합

- 이유:
  1. 선택 시 Page에서 draft를 생성하는 논리와 일관성 유지
  2. Editor는 상태 구조를 모르고 patch만 전달하도록 제한해  
     상위에서 상태 관리를 전담하는 설계를 유지하기 위함
  3. Editor가 상태 구조에 의존하지 않도록 분리해  
     다른 데이터 구조에서도 재사용 가능하게 만들기 위함

- 고려했지만 선택하지 않은 이유:
  - Editor에서 draft를 완성할 경우,
    상태 구조를 직접 다루게 되어
    컴포넌트 책임 분리가 불분명해지고 재사용성이 떨어짐
