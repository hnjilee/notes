## 의사결정

### selectedCategory 없을 때 타입 처리

- 고민:
  - 선택된 카테고리가 없는 경우(즉 전체를 표시해야 할 때)를 나타내는 타입 설정

- 선택지:
  1. `'' / null`
  2. `'ALL'`

- 판단 기준:
  - 명시성/명확성

- 결정:
  - 필터 상태를 명시적으로 나타내는 상수 `'ALL'` 사용

- 이유:
  - `if (!selectedCategory) return true;` 방식은  
    falsy 기반의 암묵적 조건이라 명확성과 설명력이 떨어짐
  - `if (selectedCategory === 'ALL')` 방식은  
    의미를 명시적으로 표현해서 명확하고 버그 가능성이 낮음

- 트레이드오프:
  - 실제 데이터에는 존재하지 않는 값 추가
  - categories 생성 시 `'ALL'`을 별도로 포함해야 함
  - Editor에서는 사용하면 안 되는 값이라 사용 범위 관리 필요

- 맥락:
  - 현재 단계에서는 데이터 단순성보다 명확성과 설명력이 더 중요하다고 판단

- 개선 여지:
  - 아예 카테고리를 상수로 분리하면 오타를 방지하고 유지 보수가 용이해질 수 있을 것
