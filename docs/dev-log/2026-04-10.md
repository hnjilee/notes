## 유효성 검사

### 현재 문제

- 노트 저장 시, `category: ''` 저장 가능  
  → categories 파생 시 `''` 포함  
  → UI에 `''` 렌더링

### 해결 전략

1. 저장 시점에 잘못된 데이터 차단
2. categories 생성 시 필터링

## 의사결정

### 유효성 검사 위치

- 고민:
  - 유효하지 않은 카테고리(`''`)를 걸러낼 위치

- 선택지:
  1. Editor에서 입력 시점에 막기
  2. Page에서 저장 시점에 막기
  3. categories 생성에서만 필터링

- 판단 기준:
  - 데이터 무결성

- 결정:
  - `handleSaveNote`에서 잘못된 데이터 차단
  - (추가로) categories 생성 시 필터링

- 이유:
  - Editor는 UI 레벨이라 로직 우회 가능성이 있고, 데이터 저장을 직접 담당하지 않기 때문에 데이터 무결성을 보장하는 위치로 적절하지 않다고 판단
  - 저장 단계에서는 잘못된 데이터가 state에 들어가는 것을 확실하게 방지할 수 있고, 보다 복잡한 조건도 추가할 수 있음
  - categories 생성에서만 필터링하면, 이미 잘못된 데이터가 state에 저장되어 다른 로직에 영향을 줄 수 있음

- 트레이드오프:
  - validation 로직이 Page에 집중되어 책임이 커짐
  - UI 레벨에서 즉각적인 피드백이 부족함 (alert 등으로 보완 필요)

- 맥락:
  - 로직의 단순성보다 데이터 보호가 더 중요하다고 판단

- 개선 여지:
  - 추후에 Custom Hook이나 상태 관리 라이브러리로 로직 분리 가능
