## 의사결정

### API 중복 호출로 인한 에러 UI 테스트 방해

- 고민:
  - React StrictMode 환경에서 useEffect가 두 번 실행되면서 API 요청이 중복 호출되고,
  - 랜덤 에러 테스트 시 성공 + 실패가 동시에 발생하는 문제가 발생함
  - 이로 인해 에러 UI 검증이 어려워짐

- 선택지:
  1. StrictMode를 유지하고 중복 호출을 그대로 둔다.
  2. useRef를 사용해 첫 호출만 실행되도록 제어한다.

- 판단 기준:
  - 비동기 흐름 검증의 정확성
  - React 동작 방식에 대한 이해 유지 여부

- 결정:
  - 개발 시에는 useRef를 사용해 중복 호출을 방지

- 이유:
  - 현재 단계에서는 로딩/에러 UI 흐름을 명확하게 검증하는 것이 더 중요하다고 판단
  - StrictMode로 인해 요청이 두 번 발생하면 에러 테스트 결과가 비일관적으로 나타나 검증이 어려움

- 트레이드오프:
  - React StrictMode의 의도(side effect 검증)를 일부 우회하게 됨
  - 실제로는 중복 호출 상황에서도 안전하게 동작하도록 설계하는 것이 더 바람직함

- 영향:
  - useEffect 내부에서 API 호출이 한 번만 실행되도록 제한
  - 개발 환경에서의 테스트 안정성은 확보되지만, StrictMode의 검증 효과는 일부 감소

- 결론:
  > 개발 단계에서는 테스트 안정성을 위해 호출을 제어하되, 실제 설계에서는 중복 호출에도 안전한 side effect 구조를 만드는 것이 더 중요하다.
