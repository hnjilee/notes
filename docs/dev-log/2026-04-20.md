### 해결 전략

useNotes 커스텀 훅으로 상태와 로직을 UI와 분리

> 컴포넌트는 무엇을 보여줄지만 알고, 어떻게 동작하는지는 모른다.

```
NotePage → 렌더링 + 이벤트 연결
useNotes → 상태 + 데이터 흐름
```

👉 핵심:

- **UI에서 필요한 것만 expose**
- 내부 구현 숨김

### 영향

- UI 담당으로 **컴포넌트 책임이 명확**해짐
- 확장이 용이해짐
  - 다른 페이지에서 상태/로직 **재사용** 가능
  - UI 교체 가능
- 데이터 흐름을 한 곳에서 관리할 수 있음
